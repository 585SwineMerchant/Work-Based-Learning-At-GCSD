package, which website pages support it, and what remaining design/export

## Remaining Steps Before the Event

- [ ] Generate the QR code image from the live URL and test it (see
      `qr-code-instructions.md`)
- [ ] Lay out and print the display board panels
- [ ] Source/produce the table covering
- [ ] Design and print both flyers (student/family and employer/partner)
- [ ] Build and export the attention-getter slideshow, load onto a device
- [ ] Do a full dry run of the assembled table before the actual event
- [ ] Confirm no source/course-context documents are visible, printed, or
      referenced anywhere in the physical table materials
- [ ] Reconfirm all copy still says "proposed" / "course project draft" —
      do not remove draft language before district approval exists

## Source-Handling Reminder

None of the physical table materials should reference, display, or link the
course PDFs, Module 1 assignment documents, internal templates, or other
source/context files kept in the project folder (these remain out of the
public website and out of git — see `.gitignore`). Everything in this
package was written directly from the program context already established
for the public site.
